### Lessons Learned - OpenClaw Cron Reminders
**Correct syntax for `openclaw message send` in cron jobs:**
```bash
openclaw message send --channel telegram --account default -t '7652382077' -m '🌿 Message text'
```

**Key points:**
- Requires `--channel telegram` and `--account default`
- Target (`-t`) is just the user ID, not the full channel prefix
- Message (`-m`) requires the `-m` flag, not positional argument
- Previous attempts failed because syntax changed from older versions

**Working cron example:**
```
30 10 3 4 * export HOME=/home/groot13-pi && /home/groot13-pi/.npm-global/bin/openclaw message send --channel telegram --account default -t '7652382077' -m '🌿 Test alert'
```

Tested successfully at 10:30 AM - message received via Telegram.
